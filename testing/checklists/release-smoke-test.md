# Release Smoke Test

Run this before publishing changes.

## Core navigation

- [ ] Open `index.html` and verify hero, header, and footer render.
- [ ] Verify top navigation links open the correct pages.
- [ ] Verify footer links open the correct pages.

## Service flows

- [ ] From `services.html`, go to request flow and confirm page opens.
- [ ] Submit the service request form with valid inputs.
- [ ] Confirm redirect lands on `thank-you.html` with expected query type.

## Contact flow

- [ ] Open `contact.html` and submit with valid data.
- [ ] Confirm success message or expected redirect behavior.
- [ ] Confirm required field validation appears for empty required fields.

## Responsive checks

- [ ] At mobile width (~390px), verify no major overlap or clipped text.
- [ ] At tablet width (~768px), verify sections stack and spacing is readable.
- [ ] At desktop width (~1440px), verify layouts align and remain balanced.

## Link integrity

- [ ] Verify no obvious broken internal links.
- [ ] Verify external links open correctly (where expected).

## Final pass

- [ ] Check browser console for JavaScript errors on key pages.
- [ ] Confirm brand visuals and typography match expected style.
